# 01 — Figma Project Structure (Workspace + Pages)

## Workspace Folder Structure (mandatory)
**Food Delivery System**
- 01 Design System
- 02 Consumer Website
- 03 Customer Mobile App
- 04 Restaurant Dashboard
- 05 Delivery Partner App
- 06 Admin Panel
- 07 Motion Assets
- 08 Landing Pages
- 09 Prototype Flows
- 10 UI Components

> Add optional sub-folders inside each section, but do not rename the top-level sections.

## Naming Conventions
- Use numeric prefixes for ordering:
  - `01-DesignTokens`
  - `02-Components`
  - `03-Motion`
- Page titles:
  - `DS - Tokens - Colors (Light/Dark)`
  - `APP - Customer - Auth - Login`
- Frames:
  - `ScreenName / State / Breakpoint`

## Breakpoints (Figma)
- Mobile:
  - 360x800 (small)
  - 390x844 (iPhone)
  - 414x896 (large)
- Web:
  - 390x844 (mobile web preview)
  - 1440x900 (desktop)

## Prototype Organization
Under **09 Prototype Flows** create pages for:
- `Flow - First Time User`
- `Flow - Checkout to Success`
- `Flow - Live Tracking Timeline`
- `Flow - Dispute / Refund Entry`

Each flow page must include:
- Entry point(s)
- All intermediate screens
- Primary CTA at each step
- Failure states (at least one per flow)
